================================================================================
THE 5 GATES - HOW THEY WORK
================================================================================

GATE 1: ML ACCURACY CHECK
───────────────────────────────────────────────────────────────────────────────
Location: CheckAndExecuteVerdictAutoEntry() line ~32075
Condition: mlAccuracy = g_lastAIConfidence * 100.0
Action: BLOCK if mlAccuracy < 75%
Log: "🔴 GATE 1 BLOQUÉ: ML accuracy X% < 75%"

Rationale:
- AI server confidence below 75% = unreliable model predictions
- Blocks trades when model accuracy is below acceptable threshold
- User sees immediate feedback when blocked


GATE 2: CONFIDENCE THRESHOLD CHECK
───────────────────────────────────────────────────────────────────────────────
Location: CheckAndExecuteVerdictAutoEntry() line ~32088
Condition: currentConfidence = g_finalVerdict.finalConfPct
Action: BLOCK if currentConfidence < 80%
Log: "🔴 GATE 2 BLOQUÉ: Confidence X% < 80%"

Rationale:
- Only enter on HIGH confidence signals (80%+)
- Signals below 80% are too weak = coin flip probability
- Dramatically reduces false entry frequency


GATE 3: MARKET STATE CHECK (ASIAN SESSION)
───────────────────────────────────────────────────────────────────────────────
Location: CheckAndExecuteVerdictAutoEntry() line ~32097
Condition: currentHour >= 22 || currentHour < 8 (London time)
Action: BLOCK during Asian session (22:00-08:00 London)
Log: "🔴 GATE 3 BLOQUÉ: Session Asiatique"

Rationale:
- Asian hours = low liquidity + unpredictable price action
- News events + reduced volume = higher loss probability
- Market structure breaks down = valid signals become false


GATE 4: VOLATILITY CHECK
───────────────────────────────────────────────────────────────────────────────
Location: CheckAndExecuteVerdictAutoEntry() line ~32106
Condition: atrValue = iATR(_Symbol, PERIOD_H1, 14)
Action: BLOCK if ATR < 0.0005 * _Point
Log: "🔴 GATE 4 BLOQUÉ: Volatilité insuffisante"

Rationale:
- Low volatility = choppy, range-bound market
- Too much noise = false breakouts = losses
- Minimum ATR ensures sufficient move available


GATE 5: SIGNAL QUALITY SCORE
───────────────────────────────────────────────────────────────────────────────
Location: CheckAndExecuteVerdictAutoEntry() line ~32118
Calculation: Quality = (Confidence × 0.40) + (Score × 25 or 15) + (ML × 20 or 10) + (Vol × 15 or 5)
Action: BLOCK if Quality < 75%
Log: "🔴 GATE 5 BLOQUÉ: Signal quality score X%"

Rationale:
- Comprehensive quality scoring combining all factors
- Weights confidence (40%), direction score (25%), ML accuracy (20%), volatility (15%)
- Only best signals pass threshold


SUCCESS LOG: When all gates pass
───────────────────────────────────────────────────────────────────────────────
Log: "✅ TOUS LES GATES PASSÉS - Signal qualifié (quality=X%)"

This confirms:
- ML accuracy >= 75%
- Confidence >= 80%
- Market hours (not Asian session)
- Sufficient volatility
- Quality score >= 75%

Trade executes immediately after this message.

================================================================================
THROTTLING MECHANISM
================================================================================

Gate messages are throttled to avoid log spam:
  - Gate 1 (ML): Log every 30s max
  - Gate 2 (Conf): Log every 30s max
  - Gate 3 (Asian): Log every 60s max
  - Gate 4 (Vol): Log every 30s max
  - Gate 5 (Quality): Log every 30s max

This prevents the Expert tab from being flooded with identical messages
while still showing when gates are blocking.

Example: If Gate 1 blocks 10 times in 30s, you see the message 1x.
After 30s, you see it again if still blocking.

================================================================================
TECHNICAL DETAILS
================================================================================

Global Variables Used:
  - g_lastAIConfidence (0..1 unit) → converted to %
  - g_finalVerdict.finalConfPct (0..100%)
  - Hour() function (current hour)
  - iATR() indicator
  - _Point symbol constant

Data Flow:
  1. Signal detected (Verdict or Spike)
  2. Enter CheckAndExecuteVerdictAutoEntry() or CheckAndExecuteSpikeTrade()
  3. Execute all 5 gates in sequence
  4. If any gate fails → RETURN (block trade)
  5. If all gates pass → CONTINUE to trade execution

Backward Compatibility:
  - No config changes required
  - Gates activate automatically
  - Existing parameters respected
  - No breaking changes

CUSTOMIZATION

To adjust gate thresholds, edit these values in the function:

Gate 4: Change "0.0005 * _Point" to your volatility threshold
  double minATR = 0.0005 * _Point;  →  your minimum

Gate 5: Change quality calculation weights
  Currently: Confidence=40%, Score=25%, ML=20%, Vol=15%
  Edit the formula to your preference

TROUBLESHOOTING
================================================================================

Q: I'm seeing too many "GATE 1 BLOQUÉ" messages
A: ML accuracy is consistently below 75%. Either:
   - Wait for AI model to recalibrate (may take 24h)
   - Lower threshold to 70% (edit Gate 1)
   - Verify AI server is healthy: http://127.0.0.1:8000/health

Q: No trades executing at all
A: Check which gate is blocking most (look at logs). Examples:
   - Mostly Gate 3 (Asian): Trade during 8am-10pm instead
   - Mostly Gate 2 (Confidence): Lower confidence to 75%
   - Mostly Gate 5 (Quality): Adjust weights in formula

Q: Gates not logging at all
A: Verify:
   1. SMC_Universal.mq5 compiled successfully (0 errors)
   2. Expert tab is open (Alt+L)
   3. EA is attached to chart (check bottom-right)
   4. EnableTrading = true
