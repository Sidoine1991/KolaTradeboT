================================================================================
HOW IT WORKS NOW
================================================================================

SIGNAL DETECTED:
   ↓
   PassAllGates() checks local confidence (Gate 2)
   ↓
   ValidateWithAIServer() called
   ↓
   TRY LOCAL (http://127.0.0.1:8000):
      If response received & valid:
         • Set g_aiConnected = true
         • Set g_aiConnectionStatus = "LOCALHOST"
         • Extract confidence from JSON
         • Display on dashboard
         ✓ DONE
   ↓
   If local fails, TRY RENDER (https://kolatradebot.onrender.com):
      If response received & valid:
         • Set g_aiConnected = true
         • Set g_aiConnectionStatus = "RENDER"
         • Extract confidence from JSON
         • Display on dashboard
         ✓ DONE
   ↓
   If both fail:
      • Set g_aiConnected = false
      • Set g_aiConnectionStatus = "FAILED"
      • Display error on dashboard
      • Log failure

EVERY 30 SECONDS:
   ↓
   CheckAIServerHealth() runs
   ↓
   Tests local server → If OK, mark LOCALHOST
   ↓
   If local fails, tests Render → If OK, mark RENDER
   ↓
   If both fail, mark FAILED
   ↓
   Updates dashboard automatically

TROUBLESHOOTING

Problem: Dashboard shows "DISCONNECTED"
Solution:
   1. Check if http://127.0.0.1:8000 is running
   2. If not, server will fallback to Render automatically
   3. Verify internet connection for Render server
   4. Check logs for error messages

Problem: Dashboard shows "FAILED" (both servers down)
Solution:
   1. Start local AI server: python ai_server.py
   2. Verify Render server is up: https://kolatradebot.onrender.com
   3. Check network connectivity
   4. Robot will resume trading when connection restored
