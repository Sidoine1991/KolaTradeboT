PROBLEM IDENTIFIED
================================================================================

SpikeRider was trading in counter-trend because:

1. JsonExtractBool defaulted to TRUE when keys were missing
   → When "/spike-tv-state" didn't have "counter_trend" field,
     g_tvCounterTrend was set to true, blocking ALL trades (false positive)

2. Counter-trend verdict wasn't properly respected
   → Even when GOM returned counter_trend=true, it was ignored

3. No staleness check on TradingView data
   → EA traded on 2+ minute old data

4. Missing safe defaults
   → When AI server offline, variables in undefined state

================================================================================
FIXES APPLIED
================================================================================

✓ FIX #1: JsonExtractBool (LINE 625-632)
  - Now defaults to FALSE (not TRUE) when key missing
  - Properly parses true/false values
  - Safe fallback for invalid values

✓ FIX #2: Counter-Trend Detection (LINE 484-509)
  - Primary layer: GOM verdict "counter_trend" field
  - Secondary layer: Structure confirmation (M15/H1 bearish/bullish)
  - Fail-safe: Multi-condition blocking

✓ FIX #3: Staleness Validation (LINE 853-862)
  - Rejects TV data older than 120 seconds
  - Prevents trading on stale market context

✓ FIX #4: Safe Defaults (LINE 803-862)
  - Every TV variable initialized with explicit defaults
  - "NEUTRAL" for directions, "neutral" for structures
  - FALSE for boolean flags, 0 for numeric values

✓ FIX #5: Enhanced Logging (LINE 844-860)
  - Logs now show "verdict_CT=BLOQUE|ok"
  - Shows exact reason for blocks
  - Better diagnostics when AI server offline
