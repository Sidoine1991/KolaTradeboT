WHAT WILL BE DRAWN ON THE CHART
================================================================================

1. GOM ENTRY LEVELS (Horizontal Lines)
   Color: GREEN for BUY levels, RED for SELL levels
   Style: Solid line, width 2
   Label: "BUY (N touches)" or "SELL (N touches)"

   What they represent:
   - Support levels where price has touched N times (within ATR tolerance)
   - Resistance levels where price has touched N times
   - Ideal entry zones for divergence signals


2. ORDER BLOCK ZONES (Colored Rectangle)
   Color: GREEN rectangle for BUY blocks, RED for SELL blocks
   Transparency: 30% (semi-transparent)
   Label: "OB BUY" or "OB SELL" in center

   What it represents:
   - Liquidation zone where smart money trapped traders
   - Price rejection candle patterns (after down, price bounces up = BUY OB)
   - Confirmed when body size >= 40% of total range


3. SIDO PATTERNS (Dashed Lines)
   Color: ORANGE dashed lines at level1 and level2
   Style: Dashed line, width 1
   Label: "DOUBLE_TOP" or "DOUBLE_BOTTOM"

   What they represent:
   - Double peaks at similar prices (within 35% ATR tolerance)
   - Double troughs at similar prices (within 35% ATR tolerance)
   - Confirmation for trend reversals


4. DASHBOARD (Text Box - Bottom Left)
   Shows:
   - Symbol and timeframe
   - Status: ACTIVE
   - AutoTrading: ON/OFF
   - Position count / Max allowed
   - Trades today / Max per day
   - Last signal direction and confidence
   - GOM level count
   - Order Block status
   - SIDO pattern type

================================================================================
HOW TO ENABLE/DISABLE DRAWINGS
================================================================================

GOM Levels:
   Input: EnableGOMEntryLevels = true
   Timeframes to show:
   - ShowM1Levels = true (1-minute GOM)
   - ShowM5Levels = true (5-minute GOM)
   - ShowM15Levels = true (15-minute GOM)
   - ShowM30Levels = false (30-minute GOM - disabled by default)
   - ShowH1Levels = true (1-hour GOM)

Order Blocks:
   Input: EnableOrderBlockDetection = true
   Configure:
   - OBLookbackBars = 50 (scan last 50 H1 bars)
   - OBMinBodyPercent = 0.4 (candle body must be 40%+ of range)

SIDO Patterns:
   Input: EnableSIDO = true
   Configure:
   - SIDOBarsToAnalyze = 300 (scan last 300 bars)
   - SIDOToleranceATRPercent = 35.0 (peaks/troughs within 35% ATR of each other)

TROUBLESHOOTING DRAWINGS
================================================================================

Q: I don't see any drawings on the chart
A: Check these in order:

   1. Is the EA attached?
      → Chart title should show "Expert Advisors: Divergence_Robot_With_GOM"

   2. Are the enable flags set correctly in inputs?
      → EnableGOMEntryLevels = true
      → EnableOrderBlockDetection = true
      → EnableSIDO = true

   3. Check the dashboard (bottom-left text)
      → Should show: Status: ACTIVE
      → Should show: Positions: 0/3

   4. Wait longer (30-60 minutes)
      → GOM detection needs time to scan history
      → Increase BarsForTouchCount to 300 to scan more history

   5. Refresh the chart (F5)
      → Clears drawing cache and redraws everything

Q: I see drawings but they disappear after a while
A: This is normal. Drawings are:
   - Deleted and redrawn every tick (part of update cycle)
   - Maintained as long as detection criteria are met
   - Removed if parameters no longer match

Q: The dashboard text is hard to read
A: Try:
   - Maximize the chart window
   - Set chart background to light color (View → Chart Options)
   - Or adjust text via Comment() function (currently shows in monospace)

Q: GOM lines are all over the place (too many or too few)
A: Adjust these parameters:
   - BarsForTouchCount: 150 → 300 (scan more history)
   - TouchZoneATRPercent: 25.0 → 30.0 (wider touch zone = more levels)
   - TouchesForMaxWidth: 8 → 5 (need fewer touches to confirm)

Q: Order Block zone never appears
A: Check:
   - EnableOrderBlockDetection = true
   - OBLookbackBars = 50 (scan range)
   - OBMinBodyPercent = 0.4 (might be too strict)
   - Lower OBMinBodyPercent to 0.3 for more detections

Q: No SIDO patterns showing
A: Adjust:
   - EnableSIDO = true
   - SIDOToleranceATRPercent: 35.0 → 50.0 (more tolerance)
   - SIDOBarsToAnalyze: 300 → 500 (scan more history)
