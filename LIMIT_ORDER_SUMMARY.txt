ORDER DETAILS

Type: LIMIT BUY or LIMIT SELL (not market order)
Entry Price: EMA M1 Fast level (marked on chart)
Stop Loss: OB boundary ± 20 pips
TP1 (First Target): Entry ± (ATR × 0.5)
TP2 (Second Target): Entry ± (ATR × 1.0) [saved for manual close]
TP3 (Third Target): Entry ± (ATR × 1.5) [saved for manual close]

Filling Type: IOC (Immediate Or Cancel)
- If exact price not available, order rejected automatically
- Prevents partial fills at bad prices

Commission: Standard
Volume: Optimal lot size (calculated by risk management)

VERDICT TIER FILTERING

Score Range         Verdict              M1/M5 Required   Entry
────────────────────────────────────────────────────────────────
|Score| < 0.35      WAIT/HOLD           ❌               No order
0.35-0.65          GOOD BUY/SELL       ✓ (GOOD)         LIMIT @ EMA M1
≥ 0.65             PERFECT BUY/SELL    ✓ (PERFECT)      LIMIT @ EMA M1

Key Point: ONLY GOOD or PERFECT verdicts trigger limit orders

═══════════════════════════════════════════════════════════════════════════════

TRADING FLOW

1. OnTick() runs continuously
   └─ Detects OB+CHOCH on chart

2. DisplayComprehensiveVerdict() updates
   └─ Shows M1/M5/H1 directions
   └─ Calculates final score
   └─ Displays verdict (WAIT/GOOD/PERFECT)

3. CheckAndExecuteOTEEntry() checks conditions
   └─ Is M1 or M5 directionally correct?
   └─ Is |Score| >= 0.35?
   └─ Does H1 confirm?
   └─ All yes? → Place limit order

4. Order placed at EMA M1 Fast level
   └─ LIMIT BUY/SELL waiting for price to touch

5. Price touches entry level
   └─ Order fills automatically
   └─ Position now open with SL + TP1

6. Position management
   └─ TP1 hits → Close 33% (automatic via trade.Buy/Sell TP1)
   └─ TP2 hits → Close 33% (manual via saved g_lastTP2)
   └─ TP3 hits → Close 33% (manual via saved g_lastTP3)
   └─ SL hits → Close all (automatic)
