================================================================================
TROUBLESHOOTING GUIDE
================================================================================

Issue 1: "GHOST: signal absent or too old"
  Cause: gom_signal.json not being updated
  Solution:
    1. Verify gom_verdict_poller.py is running
    2. Check data/gom_signal.json modification time (should be < 60s ago)
    3. Verify TradingView Desktop has GHOST_OrderFlow.pine attached
    4. Check MT5 logs for detailed error

Issue 2: No entries after 30 minutes
  Cause: GHOST filter too strict (quality < 40%) or no spikes detected
  Solution:
    1. Lower InpGHOSTMinQuality to 30.0
    2. Check InpSpikeBodyMult (default 0.50 for Boom500)
    3. Verify chart showing spike candlesticks
    4. Review Expert log for "GHOST quality < 40%" rejections

Issue 3: Dashboard GHOST panel not visible
  Cause: InpUseGHOST = false or g_ghost.valid = false
  Solution:
    1. Verify InpUseGHOST = TRUE in EA inputs
    2. Check Expert log for "[GHOST] verdict=..." messages
    3. Verify gom_signal.json has "ghost_available": true
    4. Restart EA and monitor next tick

Issue 4: False "GHOST buyers X% oppose SELL" rejections
  Cause: Strong divergence between GOM and GHOST sentiment
  Solution:
    1. This is INTENTIONAL (confluence gate working)
    2. Review entry timing (adjust InpWindowStart/End if too early)
    3. Monitor win rate (should improve despite fewer trades)

Issue 5: Compilation failed (if re-compiling)
  Solution:
    1. Verify GOMVerdict.mqh is in Include folder
    2. Check for syntax errors in custom modifications
    3. Ensure MetaEditor version is current (Build 4640+)
    4. Run "Tools → Compile" manually in MetaEditor
