## What's Fixed

### 🔧 Bug Fix #1: JsonExtractBool Default Value
**Issue:** Missing JSON keys defaulted to `true` (dangerous)  
**Fix:** Now defaults to `false` (safe)  
**Impact:** Stops false-positive counter-trend blocks

### 🔧 Bug Fix #2: GOM Counter-Trend Blocking
**Issue:** `/spike-tv-state` verdict ignored  
**Fix:** Multi-layer detection (verdict + structure)  
**Impact:** Respects TradingView market sentiment

### 🔧 Bug Fix #3: Staleness Validation
**Issue:** Traded on 2+ min old data  
**Fix:** Rejects data > 120 seconds old  
**Impact:** Safer, more timely entries

### 🔧 Bug Fix #4: Safe Defaults
**Issue:** Undefined behavior when AI server down  
**Fix:** Explicit safe defaults for all variables  
**Impact:** Stable operation even with server latency

### 🔧 Enhancement #5: Better Logging
**Issue:** No visibility into GOM verdict state  
**Fix:** Logs now show `verdict_CT=BLOQUE|ok`  
**Impact:** Better diagnostics for troubleshooting

## Support & Troubleshooting

### Issue: No GOM verdict in logs
**Check:**
1. AI server running: `curl http://127.0.0.1:8000/health`
2. `/spike-tv-state` returns: `"counter_trend": true/false`
3. EA parameter: `InpUseTVBridge=true`

### Issue: Too many "données expirées" errors
**Solution:**
1. Check AI server latency
2. Increase `InpTVBridgeMaxAgeSec` to 30-40 (if needed)
3. Verify TradingView MCP connection is stable

### Issue: All trades blocked
**Check:**
1. Verify GOM verdict is actually `true` (not all trades should block)
2. Check market structure (M15/H1) matches GOM verdict
3. Review AI server logic for false positives

### Rollback
```bash
# Quick rollback to v5.03
cp SpikeRiderEA.ex5.backup SpikeRiderEA.ex5
# Reload MT5
```
