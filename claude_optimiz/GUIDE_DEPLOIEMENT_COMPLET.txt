2️⃣  INSTALLATION OLLAMA (Local LLM)
═════════════════════════════════════════════════════════════════════════════════

ÉTAPE A: Télécharger Ollama
  1. Aller sur: https://ollama.ai
  2. Télécharger pour votre OS (Windows/Mac/Linux)
  3. Installer avec les options par défaut

ÉTAPE B: Lancer Ollama et télécharger modèle
  
  Windows:
    - Ollama se lance automatiquement au démarrage
    - Ouvrir CMD:
      $ ollama pull mistral
      $ ollama serve
  
  Linux/Mac:
    $ ollama pull mistral
    $ ollama serve

  ⏳ LE PULL PREND 5-10 MINUTES (Mistral = 4.1GB)
  ⏳ LAISSER TOURNER (ollama serve écoute sur 127.0.0.1:11434)

ÉTAPE C: Vérifier Ollama est prêt
  Ouvrir navigateur:
  → http://127.0.0.1:11434/api/tags
  
  ✅ Si JSON avec liste de modèles = Ollama est OK ✅

⚠️  IMPORTANT: Laisser ollama serve tourner TOUT LE TEMPS

6️⃣  TROUBLESHOOTING - RÉSOUDRE LES PROBLÈMES
═════════════════════════════════════════════════════════════════════════════════

❌ PROBLÈME: "AI Server OFFLINE" dans MT5
─────────────────────────────────────────
SOLUTION:
  1. Vérifier serveur lancé:
     cmd> curl http://127.0.0.1:8000/health
  
  2. Vérifier port 8000:
     Windows: netstat -ano | findstr :8000
     Linux: lsof -i :8000
  
  3. Vérifier firewall:
     - Autoriser port 8000
     - Débrancher firewall test
  
  4. Vérifier URL dans MT5:
     - Doit être: http://127.0.0.1:8000
     - PAS: http://localhost:8000

❌ PROBLÈME: "OLLAMA INDISPONIBLE" dans serveur
──────────────────────────────────────────────
SOLUTION:
  1. Vérifier Ollama tourne:
     cmd> ollama serve
  
  2. Vérifier port 11434:
     Windows: netstat -ano | findstr :11434
     
  3. Vérifier modèle installé:
     cmd> ollama pull mistral
  
  4. Tester connexion:
     cmd> curl http://127.0.0.1:11434/api/tags

❌ PROBLÈME: Compilation erreur MQ5
──────────────────────────────────
SOLUTION:
  1. Ouvrir MetaEditor
  2. Tools → Options → Compilation
  3. Vérifier chemins des includes
  4. Compiler seul (F5) puis exécuter
  5. Vérifier pas de WebRequest désactivé

❌ PROBLÈME: Positions n'ouvrent pas
───────────────────────────────────
SOLUTION:
  1. Vérifier RiskPerTrade = 50.0 (minimum)
  2. Vérifier solde du compte >$500
  3. Vérifier pas au-delà de MaxPositions
  4. Lancer robot en mode "Visual Backtest" test
  5. Vérifier logs erreurs dans EA

❌ PROBLÈME: Latence IA élevée (>3000ms)
────────────────────────────────────────
SOLUTION:
  1. Vérifier CPU usage:
     - Fermer applications
     - Vérifier pas de scan antivirus
  
  2. Ajuster timeout:
     - GOM_KOLA: AI_TIMEOUT_MS = 3000
     - SMC: AI_TIMEOUT_MS = 3000
  
  3. Réduire MaxBarsToAnalyze:
     - De 300 à 150
  
  4. Redémarrer Ollama:
     - Tuer processus ollama serve
     - Relancer: ollama serve

❌ PROBLÈME: Confiance IA toujours 0
───────────────────────────────────
SOLUTION:
  1. Vérifier MIN_IA_CONFIDENCE:
     - Réduire de 0.55 à 0.50
  
  2. Vérifier tendance:
     - Robot doit détecter UPTREND/DOWNTREND
     - EN cas de NEUTRAL, fallback s'active
  
  3. Vérifier Ollama répond:
     cmd> curl http://127.0.0.1:11434/api/generate \
          -d '{"model":"mistral","prompt":"Hello"}'
  
  4. Vérifier logs serveur:
     - Regarder stdout du python
     - Doit voir "✅ OLLAMA: SYMBOL → BUY (0.65)"
